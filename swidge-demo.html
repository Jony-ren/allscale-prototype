这是第二部手机扫描 QR Code 之后，在钱包内置浏览器中打开的界面（有浏览器 Chrome）。
改动 本次需要做的产品改动
范围：整个 Swidge 模块（即所有跨链相关的功能）+ Checkout 主金额块的 USDT/USDC toggle
具体包括：
• Checkout 主金额块新增 USDT/USDC toggle pill
• 「Pay with other tokens」入口 + Token / Chain 选择列表 + Quote 报价 + Bridging 等待页等所有跨链相关流程
• Dashboard Settings 下的 Swidge 开关与「Supported chains」展示卡
• 新路由规则：6 条 EVM 链 + USDT/USDC → 直转免 Swidge；其余走 Swidge
⚠️ 旧版的 Master Chain 单选已删除，商家不再需要配置主链。
Demo 用 Demo 偷懒的地方 — 不要照搬
为了 demo 美观做了简化，但真实代码要保留原有完整逻辑：
• Checkout 「Details」卡只展示 3 行（Order ID / Amount / Merchant）—— 真实环境保留原本所有字段（含 Exchange rate / Created / Expires 等），不要按 demo 删行
• chainData 里的 fees / balance / rate 都是 hardcode 样例 —— 真实通过 Rhino.fi / LI.FI 实时询价
• 「Bridging to BNB Chain」状态文案是单一样例 —— 真实落点链由路由按成本动态选（fallback BNB）
• 顶部 USDT/USDC toggle 仅 Checkout 主页面切换 —— 真实需联动 QR / Confirm / 报价等下游金额
• Demo 仅展示 4 条链的 Token，真实需覆盖完整 6 条 EVM 链上的 USDT 与 USDC
